### 배열포인터 / 포인터배열

1. 배열포인터 (Pointer to Array)
    - 정의
        - 배열 전체를 가리키는 포인터

    - 선언
        - int (*p)[5];
        - → p는 "int 5개짜리 배열을 가리키는 포인터"

    - 예시
        - int arr[5];
        - int (*p)[5] = &arr;

    - 특징
        - p + 1 → 배열 전체 크기만큼 증가
        - 연속 메모리 구조
        - 2차원 배열 처리에 사용

2. 포인터배열 (Array of Pointers)
    - 정의
        - 포인터들을 저장하는 배열

    - 선언
        - int *p[5];
        - → p는 "int형 포인터 5개를 저장하는 배열"

    - 예시
        - int a, b, c;
        - int *p[3] = { &a, &b, &c };

    - 특징
        - p + 1 → 포인터 1개 크기 증가
        - 각 요소는 주소값 저장
        - 문자열 배열에서 많이 사용


3. 차이 비교

| 구분 | 배열포인터 | 포인터배열 |
|------|------------|------------|
| 선언 | int (*p)[5] | int *p[5] |
| 의미 | 배열을 가리킴 | 포인터를 저장 |
| 우선순위 | * 먼저 | [] 먼저 |
| 주 사용 | 2차원 배열 | 문자열 처리 |


## 핵심 정리
    - 배열포인터 = 배열을 가리키는 포인터
    - 포인터배열 = 포인터들의 배열
    - 괄호 위치가 모든 차이를 만든다



### 메모리 영역

# 전역변수 / 지역변수 / static / const / 배열 / 포인터의 메모리 위치

## 1. 프로그램 메모리 구조

프로그램이 실행되면 메모리는 크게 다음 영역으로 나뉜다.

1) Code 영역 (Text 영역)
2) Data 영역
3) Heap 영역
4) Stack 영역

---

## 2. Code 영역 (Text Segment)

- 함수 코드가 저장되는 영역
- 읽기 전용 (보통 수정 불가)
- 프로그램 시작부터 종료까지 유지

예)
    int func() { return 1; }

    printf("%p\n", func);

→ func 함수의 주소는 Code 영역 주소

---

## 3. Data 영역

프로그램 시작과 동시에 할당
프로그램 종료 시 해제

### (1) 초기화된 전역변수 / static 변수

    int global_c = 100;
    static int c;

→ Data 영역에 저장

### (2) 초기화되지 않은 전역변수

    int a;

→ BSS 영역 (Data 영역의 일부)

※ BSS : 0으로 자동 초기화되는 영역

---

## 4. Stack 영역

- 함수가 호출될 때 생성
- 함수 종료 시 자동 해제
- 지역변수 저장

예)

    int main()
    {
        int b;
        const int d;
        char arr[20] = "hi";
    }

→ b, d, arr 모두 Stack 영역

특징
- 함수 호출 시 위에서 아래로 쌓임
- 속도 빠름
- 크기 제한 존재
- Stack Overflow 발생 가능

---

## 5. Heap 영역

- 동적 메모리 할당 영역
- malloc, calloc, realloc 사용
- 직접 free 해야 함

예)

    int *p = malloc(sizeof(int));

→ Heap 영역에 생성

---

## 6. 문자열과 포인터의 차이

코드:

    char arr[20] = "hi";
    char* parr = "hong";

### arr

- 배열 자체는 Stack 영역
- "hi" 문자열 내용도 배열 안에 복사됨
- 수정 가능

### parr

- 포인터 변수는 Stack 영역
- "hong" 문자열은 읽기 전용 영역 (Code 영역 근처의 상수 영역)
- 문자열 리터럴은 수정 불가

정리

| 구분 | 포인터 변수 | 실제 문자열 |
|------|------------|------------|
| arr  | Stack | Stack |
| parr | Stack | 읽기전용 영역 |

---

## 7. 변수별 메모리 정리

| 변수 | 저장 영역 |
|------|-----------|
| func | Code |
| a (전역) | BSS |
| global_c | Data |
| b (지역) | Stack |
| c (static) | Data |
| d (const 지역) | Stack |
| arr | Stack |
| parr | Stack (문자열은 읽기전용 영역) |

---

## 8. 한눈에 그림 정리

높은 주소
-----------------
Stack  ← 지역변수
-----------------
Heap   ← malloc
-----------------
Data   ← 전역, static
BSS    ← 초기화 안된 전역
-----------------
Code   ← 함수
-----------------
낮은 주소

---

## 핵심 정리

- 전역 / static → Data 영역
- 초기화 안된 전역 → BSS
- 지역변수 → Stack
- malloc → Heap
- 함수 → Code
- 문자열 리터럴 → 읽기 전용 영역

프로그램은 이렇게 메모리 구조 위에서 동작한다.

## 배열포인터 vs 포인터배열

### 배열포인터 (Pointer to Array)
- 배열 전체를 가리키는 포인터
- 특징
    - 배열 전체 크기 단위로 이동 (`p + 1`)
    - 연속 메모리 구조
    - 2차원 배열 처리에 사용
- 선언 예시
    - `int (*p)[5];` → 5개 int 배열을 가리키는 포인터
    - 사용: `p = &arr;`

### 포인터배열 (Array of Pointers)
- 포인터들을 저장하는 배열
- 특징
    - 각 요소는 주소값 저장
    - p + 1 → 포인터 1개 크기 증가
    - 문자열 배열 처리에 자주 사용
- 선언 예시
    - `int *p[5];` → int형 포인터 5개 저장

### 차이 요약

| 구분 | 배열포인터 | 포인터배열 |
|------|------------|------------|
| 선언 | int (*p)[5] | int *p[5] |
| 의미 | 배열 가리킴 | 포인터 저장 |
| 우선순위 | * 먼저 | [] 먼저 |
| 주요 사용 | 2차원 배열 | 문자열 처리 |

### 핵심 정리
- 배열포인터 = 배열 전체를 가리키는 포인터
- 포인터배열 = 포인터들의 배열
- 괄호 위치가 의미를 완전히 바꿈

---

## 프로그램 메모리 구조

### 1. 메모리 영역 구분
프로그램 실행 시 메모리는 크게 4개 영역으로 나뉨:
1. Code 영역 (Text Segment) – 함수 코드
2. Data 영역 – 초기화된 전역/Static 변수
3. Heap 영역 – 동적 메모리
4. Stack 영역 – 지역변수

---

### 2. Code 영역
- 함수 코드가 저장되는 영역
- 읽기 전용
- 프로그램 시작부터 종료까지 유지
- 예: `func` 함수 주소 → Code 영역

---

### 3. Data 영역
- 프로그램 시작과 동시에 할당, 종료 시 해제
- 초기화된 전역/Static 변수 저장
- 초기화되지 않은 전역 변수 → BSS 영역
- 특징: 자동으로 0 초기화되는 BSS 존재

---

### 4. Stack 영역
- 함수 호출 시 생성, 종료 시 자동 해제
- 지역변수, 함수 매개변수 저장
- 특징
    - LIFO 구조, 속도 빠름
    - 크기 제한, Stack Overflow 가능

---

### 5. Heap 영역
- 동적 메모리 할당 영역
- malloc, calloc, realloc 사용
- 사용 후 반드시 free 필요
- 특징: 용량 유동적, 속도 Stack보다 느림

---

### 6. 문자열과 포인터 차이

| 구분 | 배열 변수 | 포인터 변수 |
|------|-----------|-------------|
| 메모리 | Stack | Stack (문자열은 읽기 전용) |
| 수정 가능 | 가능 | 문자열 리터럴은 불가 |

- 배열: 문자열 내용이 Stack에 복사됨 → 수정 가능  
- 포인터: 문자열 리터럴을 가리킴 → 읽기 전용

---

### 7. 변수별 메모리 정리

| 변수 | 저장 영역 |
|------|-----------|
| func | Code |
| a (전역) | BSS |
| global_c | Data |
| b (지역) | Stack |
| c (static) | Data |
| d (const 지역) | Stack |
| arr | Stack |
| parr | Stack (문자열은 읽기 전용) |

---

### 8. 한눈에 그림 정리

높은 주소
-----------------
Stack  ← 지역변수
-----------------
Heap   ← malloc 등
-----------------
Data   ← 전역, static
BSS    ← 초기화 안된 전역
-----------------
Code   ← 함수
-----------------
낮은 주소

---

### 핵심 정리
- 전역/Static → Data 영역  
- 초기화 안된 전역 → BSS  
- 지역변수 → Stack  
- malloc → Heap  
- 함수 → Code  
- 문자열 리터럴 → 읽기 전용 영역  

- 프로그램은 위 메모리 구조 위에서 동작

### C++ 입출력 (I/O)
- 콘솔로 데이터를 주고받는 기본 방법
- 주요 객체
    - `cin` : 키보드 입력
    - `cout` : 화면 출력
    - `cerr` : 에러 출력
    - `clog` : 로그 출력
- 특징
    - 출력 버퍼 사용 → 속도 최적화 가능
    - `endl` : 줄바꿈 + flush
    - `'\n'` : 단순 줄바꿈, 속도 빠름
- 주의: 버퍼 동기화를 조절하면 성능 차이가 있음

### 함수 오버로딩
- 같은 이름의 함수를 매개변수 타입/개수로 구분하여 정의
- 조건
    - 이름 동일, 매개변수 달라야 함
    - 반환형만 다른 경우는 허용 안됨
- 장점: 코드 가독성 향상, 다양한 입력 처리 가능
- 주의: 기본 매개변수와 함께 쓰면 모호성 가능

### 디폴트 매개변수
- 함수 매개변수에 기본값 지정 → 생략 가능
- 특징
    - 오른쪽에서 왼쪽 순서로 지정
    - 선언부에만 지정하는 것이 관례
- 장점: 호출 편리, 코드 간결

### 함수 인라인화
- 호출 시 정의부 코드가 삽입되도록 권장
- 장점: 호출 오버헤드 제거 → 속도 향상
- 단점
    - 함수가 크면 코드 복사로 프로그램 크기 증가
    - 캐시 효율 저하 가능
- 핵심: `inline`은 강제가 아닌 권장 의미

### namespace
- 이름 충돌 방지, 코드 그룹화
- 특징
    - 중첩 가능
    - `using namespace`로 편리하게 사용
    - 전역 이름과 충돌 방지

### 클래스
- 변수(속성) + 함수(동작) 묶음 → 사용자 정의 타입
- 특징
    - 캡슐화 : `public`, `private`, `protected`
    - 연산자 오버로딩 : 같은 연산자 여러 정의 가능
    - 함수 오버라이딩 : 상속 시 부모 함수 재정의
- 장점: 객체 지향 프로그래밍 핵심

### 포인터와 const
- `const int *p` : 값 변경 불가
- `int *const p` : 주소 변경 불가
- `const int *const p` : 값과 주소 모두 변경 불가
- 장점: 안전한 코드 작성, 실수 방지

### 제네릭 / 템플릿
- 타입에 독립적인 함수/클래스 작성
- 특징
    - 재사용성 ↑
    - 다양한 타입 처리 가능
    - 구현은 헤더 파일에 작성
- 장점: 코드 반복 최소화

### bool 자료형
- 참/거짓 저장 (`true`, `false`)
- 특징
    - true → 1, false → 0
    - 다른 타입 자동 변환 가능
    - 최소 1 byte
- 용도: 조건 판단, 플래그 설정

### 참조자 (Reference)
- 변수의 별칭, 별도의 메모리 없이 원본 참조
- 특징
    - 값 전달 최적화
    - 상수 참조 가능 : `const int &r`
    - 포인터와 달리 NULL 참조 불가
- 장점: 코드 간결, 성능 최적화

### 핵심 정리
- 입출력 : cin/cout, 버퍼 고려
- 함수 : 오버로딩, 디폴트, 인라인
- 이름 충돌 방지 : namespace
- 객체지향 : 클래스, 오버로딩/오버라이딩
- 안전한 포인터 : const 활용
- 재사용성 : 템플릿
- 단순 참/거짓 : bool
- 값 전달 최적화 : 참조자
