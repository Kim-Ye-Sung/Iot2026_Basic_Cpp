### 배열포인터 / 포인터배열

1. 배열포인터 (Pointer to Array)
    - 정의
        - 배열 전체를 가리키는 포인터

    - 선언
        - int (*p)[5];
        - → p는 "int 5개짜리 배열을 가리키는 포인터"

    - 예시
        - int arr[5];
        - int (*p)[5] = &arr;

    - 특징
        - p + 1 → 배열 전체 크기만큼 증가
        - 연속 메모리 구조
        - 2차원 배열 처리에 사용

2. 포인터배열 (Array of Pointers)
    - 정의
        - 포인터들을 저장하는 배열

    - 선언
        - int *p[5];
        - → p는 "int형 포인터 5개를 저장하는 배열"

    - 예시
        - int a, b, c;
        - int *p[3] = { &a, &b, &c };

    - 특징
        - p + 1 → 포인터 1개 크기 증가
        - 각 요소는 주소값 저장
        - 문자열 배열에서 많이 사용


3. 차이 비교

| 구분 | 배열포인터 | 포인터배열 |
|------|------------|------------|
| 선언 | int (*p)[5] | int *p[5] |
| 의미 | 배열을 가리킴 | 포인터를 저장 |
| 우선순위 | * 먼저 | [] 먼저 |
| 주 사용 | 2차원 배열 | 문자열 처리 |


## 핵심 정리
    - 배열포인터 = 배열을 가리키는 포인터
    - 포인터배열 = 포인터들의 배열
    - 괄호 위치가 모든 차이를 만든다



### 메모리 영역

# 전역변수 / 지역변수 / static / const / 배열 / 포인터의 메모리 위치

## 1. 프로그램 메모리 구조

프로그램이 실행되면 메모리는 크게 다음 영역으로 나뉜다.

1) Code 영역 (Text 영역)
2) Data 영역
3) Heap 영역
4) Stack 영역

---

## 2. Code 영역 (Text Segment)

- 함수 코드가 저장되는 영역
- 읽기 전용 (보통 수정 불가)
- 프로그램 시작부터 종료까지 유지

예)
    int func() { return 1; }

    printf("%p\n", func);

→ func 함수의 주소는 Code 영역 주소

---

## 3. Data 영역

프로그램 시작과 동시에 할당
프로그램 종료 시 해제

### (1) 초기화된 전역변수 / static 변수

    int global_c = 100;
    static int c;

→ Data 영역에 저장

### (2) 초기화되지 않은 전역변수

    int a;

→ BSS 영역 (Data 영역의 일부)

※ BSS : 0으로 자동 초기화되는 영역

---

## 4. Stack 영역

- 함수가 호출될 때 생성
- 함수 종료 시 자동 해제
- 지역변수 저장

예)

    int main()
    {
        int b;
        const int d;
        char arr[20] = "hi";
    }

→ b, d, arr 모두 Stack 영역

특징
- 함수 호출 시 위에서 아래로 쌓임
- 속도 빠름
- 크기 제한 존재
- Stack Overflow 발생 가능

---

## 5. Heap 영역

- 동적 메모리 할당 영역
- malloc, calloc, realloc 사용
- 직접 free 해야 함

예)

    int *p = malloc(sizeof(int));

→ Heap 영역에 생성

---

## 6. 문자열과 포인터의 차이

코드:

    char arr[20] = "hi";
    char* parr = "hong";

### arr

- 배열 자체는 Stack 영역
- "hi" 문자열 내용도 배열 안에 복사됨
- 수정 가능

### parr

- 포인터 변수는 Stack 영역
- "hong" 문자열은 읽기 전용 영역 (Code 영역 근처의 상수 영역)
- 문자열 리터럴은 수정 불가

정리

| 구분 | 포인터 변수 | 실제 문자열 |
|------|------------|------------|
| arr  | Stack | Stack |
| parr | Stack | 읽기전용 영역 |

---

## 7. 변수별 메모리 정리

| 변수 | 저장 영역 |
|------|-----------|
| func | Code |
| a (전역) | BSS |
| global_c | Data |
| b (지역) | Stack |
| c (static) | Data |
| d (const 지역) | Stack |
| arr | Stack |
| parr | Stack (문자열은 읽기전용 영역) |

---

## 8. 한눈에 그림 정리

높은 주소
-----------------
Stack  ← 지역변수
-----------------
Heap   ← malloc
-----------------
Data   ← 전역, static
BSS    ← 초기화 안된 전역
-----------------
Code   ← 함수
-----------------
낮은 주소

---

## 핵심 정리

- 전역 / static → Data 영역
- 초기화 안된 전역 → BSS
- 지역변수 → Stack
- malloc → Heap
- 함수 → Code
- 문자열 리터럴 → 읽기 전용 영역

프로그램은 이렇게 메모리 구조 위에서 동작한다.
